基于Evernote + Markdown的个人信息管理方案

0 需求分析

0.1 日常需求

自己的日常信息或笔记主要有这么几类：

  • 工作记录、工作文档
  • 读书笔记、摘抄
  • 学习笔记
  • 各种信息记录，大到自己的零花钱记账、小到网上看到的CPU评测

0.2 原有状况

原本的工具主要是：

  • Evernote： 解决绝大部分工作或私人的笔记、记录、摘抄；优点在于云端同步，但是需要分享或者递送就不行，格式上也存在很多限制
  • WPS Office(Word\PPT\Excel)、 MS Visio：工作文件，功能肯定没问题，但是臃肿庞大
  • 其它零散工具（场景）：日志发布在QQ空间、内网分享文件靠QQ或邮件、办公笔记本跟家里台机之间的Bt sync同步、等等

原有方案的核心问题：

  • Evernote 以外的软件都很大很臃肿，且大部分为收费软件
  • MS office 格式文件编写的时候，很多时间和精力会分散在格式排版上而非内容本身
  • 不便于版本管理，历史版本往往只能在硬盘里留一份拷贝来备查
  • 信息分散：有文件、有知识库、有网络工具……无法统一备份、管理与查询

0.3 Evernote 是核心但不是本文重点

知识库管理必然以 Evernote 为核心，但是关于 Evernote 的介绍起码能写本书，比如《Evernote 超效率数字笔记》：

https://yunpan.cn/cSBcxLsABiD3y 访问密码 b946

0.4 为什么要用 Markdown

考虑在Evernote的基础上加入Markdown作为解决方案，主要基于 Markdown 大量的优势：

  1. 本质为 txt 纯文本
    1. 轻量、简洁
    2. 任何文本编辑器都能打开编辑查看
    3. 便于历史版本管理，不需要电脑里存一堆后面跟着V1、V2……的同名文件
    4. 编写时只需要集中精力关注内容，无需关注格式与排版，各个编辑器自带css会渲染生成格式，不满意自定义即可
    5. 跨平台支持极佳，OS X 貌似目前支持的最好, Linux/Win 也不差
  2. Markdown 格式具备广泛的兼容性与可扩展性，适用各种分享或协作场景
    1. 可导出生成： Doc、Pdf、Html、slideshow（幻灯片）、Jpg、Latex、Blog、Wiki、电子书（mobi、Epub）等等各种想得到想不到用得上用不上的文件格式；
    2. 虽然是文本格式，但是语法却支持内嵌目录导航、待办事项列表、图片、表格、数学公式、流程图、甘特图等等各种常用富文本内容
    3. Evernote、Chrome、GitHub、SublimeText、Atom等各路大佬的兼容与支持，简书 、 马克飞象等一系列小众但出色的平台或工具
    4. 让人眼花缭乱选择困难的各种md编辑器，各种或强大或奇特的功能支撑，比如写书的可以用 GitBook，写论文的有 Scrivener，而MarkEditor支持直接发布Markdown格式文章到微信……
  3. Markdown 被称为最适合程序员的书写工具，越来越多的软件的帮助文件采用md格式。所以就算单纯为了装X也得学一下啊，正好掩饰自己已经不会写代码的问题

Markdown的介绍或者说教程网上随便搜一下就行，到处都是，这里简单链两个：

  • Markdown：让书写更美好

  • Markdown简明教程：码字效率飞升80%，不懂代码也能装

1 核心工具

  • 笔记工具： Evernote 知识库的核心
  • chrome插件： Markdown Here 万能的 web 用瑞士军刀，支持在web上的一切文本输入框里使用Markdown 语法，并可进行双向的格式转换
  • web工具： wiki-in-box、hugo
  • 幻灯片工具：Landslide
  • 万能的文件格式转化工具： Pandoc （但是这个基本不用额外安装，很多都是软件里直接已经内置）
  • md编辑器：
    • typora：日常主力使用，界面清爽，有大纲视图，插入表格和图片都很方便，导出doc需自行额外安装 pandoc 包，唯一缺陷不支持流程图（但愿后续版本改进）
    • Sublime Text： 安装sublime server插件后，配合wiki-in-box使用
    • MarkdownEditor： 开源免费、导出格式齐全、语法支持完整、不过不失、最佳万金油备胎
    • MarkEditor：加入了文件管理，界面清爽，功能齐备，操作也许是为了兼顾MacOS显得有点别扭，另外有个大问题是收费软件，试用版会各种不爽

看起来不少，其实必须的也就是 Evernote 、 Markdown Here、加上一个自己最喜欢的md编辑器这三大件即可。

2 私人笔记：Evernote + Markdown Here

Evernote 目前还没有原生支持 Markdown 语法，虽然说已经加在开发团队的任务列表里了。不过看看隔壁的未知笔记、有道云笔记等等，大象这次的脚步着实慢了。 马克飞象 就是为了给 Evernote 外挂 Markdown 语法支持的第三方应用，不过因为是收费应用所以我没怎么用过……

实际上，这个问题可以用万能的 Markdown Here 插件来解决，怎么在Chrome里装这个插件就不提了，无非就是科学上网哪些事。

  1. 在 Chrome 里打开 Evernote 的 web 页面，新建笔记
  2. 按照 Markdown 语法，输入想写的东西（比如本文）
  3. 在编辑区域鼠标右击，选择“Markdown转换”

转换后的效果在客户端同步后看起来也是一样的，如果需要转换回来继续编辑，同样的方式重复操作一次即可。

用这样的方式，几乎可以在任何的 web 输入区域使用 Markdown 语法，比如 Gmail 等各种 web 邮箱，比如 QQ空间 。
