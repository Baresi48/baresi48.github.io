使用 BitBucket + Hugo 构建免费 Blog

  2016-05-17   Study Notes   Web Markdown 字数统计: 1883字 阅读时长: 4分

前言

先发半成品： baresi.bitbucket.org

优点 ：

  1. 免费！ 免费！免费！
  2. 简洁的个人二级域名；
  3. 可适当冒充 Geek ；
  4. 自带版本控制，历史版本不需要像wod一样丑陋的在文件名后加 V1\V1.1来标识了（我工作文档里有个PPT文件名上有V16我会乱说）

1 为什么要折腾这么个玩意？

1.1 要啥Blog ？

时不时总会心血来潮写点东西，一开始的MSN Space已经关了，后来短暂用段时间新浪博客后，因为平时QQ挂的多，很多东西就写了扔在QQ空间了。

但是，自从发现有Markdown这个好东西后，下意识的觉得自己的知识（文件）管理体系从此可以大一统了。

于是，需要一个支持Markdown语法的Blog的需求，立刻提上了议事日程。

当然，覆盖这个需求最理想的方法，直接用 简书 就一了百了了：界面清爽简洁、功能该有的都有、支持Markdown、有手机APP……

但是，架不住我最近比较闲，就是想折腾怎么办呢？ （当然，非要给 简书 挑刺，就是没有一个简洁域名的个人主页吧。）

1.2 为啥用 BitBucket （而不是 GitHub？）

用 GitHub 写书写 Blog 写论文写……几乎已经成为了一种时尚，随手一搜各种教程也是漫天遍野，但是 GitHub 鼓励开源，免费账户是不能建私有仓库的。作为一个想保留点隐私的穷人，觉得还是功能基本类似的 BitBucket 更适合我。

再说了，大家都用 GitHub ，小众的选择有时候更显逼格啊。

1.3 为啥用 Hugo （而不是 jekyll 或 octopress ？）

  • 必须支持Markdown；
  • Hugo 最简洁，不需要什么依赖包，生成速度也最快；
  • GitHub Pages 原生支持 jekyll ，因此用 jekyll 可以理解，但是用 BitBucket 的话，大家都在同一起跑线了
  • 不想再折腾 ruby 之类的环境（我是 python 粉）

2 构建流程

2.1 注册 BitBucket

访问 bitbucket.org ，填写用户名、邮箱和密码。

发现有人家写好的了，比我自己原来的流程还多了设置中文，直接引用吧：

  • 用Bitbucket搭建博客初探

假设注册与涉及信息如下（后面命令中会涉及到，自己根据自己的注册信息替换）

  • 用户名：myblog
  • 仓库名：myblog.bitbucket.org

额外说明一点，注册和配置完成后，本地要么装好 git ，要么装好 bitbucket 官方推荐的 SourceTree 。如果不知道选哪个，那就选后者吧，好歹有个比较直观的图形界面……

git 的使用要展开又是一个大坑，但是好在建 Blog 只需要用到 add 跟 commit 等这几个简单操作，死记硬背下来都行……

git的详细用法可以参考：Git中文教程

2.2 本地建 hugo 目录

Hugo官方主页：gohugo.io

直接上去找最新版下载就行，注意操作系统。打算在 linux 下玩这个的肯定不会需要看这种幼儿园水平的教程，以下以 windows 进行说明：

下载最新版后解压缩，里面应该是三个文件，一个exe，两个md。

hugo_0.15_windows_amd64.exe
LICENSE.md
README.md

真正有用的就那个exe文件，复制了改名成 hugo.exe 会用起来方便点。

具体使用不复杂，先放个中文手册： hugo中文手册

生成网站目录（myblog文件夹必须不存在或者为空，否则生成失败）：

$ hugo new site myblog

生成后的目录结构大致是这样的：

  ▸ archetypes/
  ▸ content/
  ▸ layouts/
  ▸ static/
    config.toml

大致上，content目录放自己写的blog文件（Markdown格式）；static放各种静态的图片；config.toml 则是全局配置文件。

具体组织结构的介绍，可以参考：hugo项目的结构

2.3 Theme 模版

默认安装的Hugo目录是没有任何皮肤模版的，需要自己下载。具体可以看官方theme模版

我选了个 hyde-y ， 安装、配置等等的说明都非常详细了。

安装（基本上所有theme的安装都是类似的）：

$ cd myblog/
$ mkdir themes
$ cd themes
$ git clone https://github.com/enten/hyde-y

2.4 开始写blog

目录管理是有点说法的，没搞明白之前，所有的文章都按照惯例放在content/post目录下比较好。

$ hugo new post/first-blog.md

然后随便找个工具编辑这个 md 文件就行。一般我用 MarkdownEditor 或 Typora ， 偶尔也用 SublimeText 。当然你用记事本也行，Markdown的优势不就是随便抓个工具都能看都能改嘛。

自动生成的md其实就是默认写进去一个文件头，具体可以参考： 文件头，略微补充几点：

  • draft 那行要么删掉， 要么把默认的 true 改为 false ，否则就是草稿，不会显示出来；
  • description 字段是摘要，如果不写。那么以后 blog 首页上会把正文打乱格式显示出来，会很丑；
  • tags 和 categories 用于文章分类的，初期不明白可以先不管，后期熟悉了建议再补加上。

2.5 本地测试

先启动本地服务来测试下效果：（如果已经在 config.toml 文件里设置了 theme=hyde-y ， 那么下面命令中的 –theme=hyde-y 可以不用加上，后面其它的命令也一样。 ）

hugo server --theme=hyde-y --buildDrafts --watch

然后在自己的浏览器里看下效果： 127.0.0.1:1313

3 发布

先生成最终的静态网页

hugo --theme=hyde-y --baseUrl="http://myblog.bitbucket.org/"

如果没有报error，那么所有静态页面都已经生成到 public 目录，将pubilc目录里所有文件 push 到 2.1节 创建的Repository的 master 分支。

$ cd public
$ git init
$ git remote add origin https://myblog@bitbucket.org/myblog/myblog.bitbucket.org.git 
$ git add -A
$ git commit -m "first commit"
$ git push -u origin master

浏览器里访问：http://myblog.bitbucket.org/ 即可。 比如我的： http://baresi.bitbucket.org/

Over!
